# officials-mk-lab

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放和展示各种独立的 HTML 页面。这些页面可能包含工具、演示、实验性内容或信息展示，彼此之间没有固定的关联。仓库本身不针对任何特定域名或网站，仅作为页面归档与发布的容器。

## 目录说明

```
├── README.md          # 本文件
├── index.html         # 仓库主页（可选）
├── pages/             # 页面归档目录
│   ├── example1/      # 示例页面文件夹
│   │   └── index.html
│   ├── example2/
│   │   └── index.html
│   └── ...
└── assets/            # 公共资源文件夹（可选）
    ├── css/
    ├── js/
    └── images/
```

- **pages/**：存放所有独立 HTML 页面的文件夹。每个页面建议单独建立一个子文件夹，以便管理关联资源。
- **assets/**：可选的公共资源文件夹，用于存放多个页面共用的样式、脚本或图片。

## 页面归档说明

- 每个页面应尽量保持独立，减少外部依赖。
- 页面文件建议命名为 `index.html`，并放置于自己的子文件夹中，便于通过路径直接访问。
- 如果页面需要引用外部资源（如字体、库），优先使用 CDN 或本地资源，但需确保来源稳定。
- 页面内容应当合法合规，不包含任何恶意代码或侵权信息。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面时会提交对应 commit。
- 欢迎通过 Issue 提出建议或报告问题，但请注意仓库定位为通用归档，不接受特定网站或商业推广请求。
- 拉取请求（Pull Request）可以提交新的页面或改进，但需经过审核后合并。提交前请确保页面内容符合上述归档说明。
